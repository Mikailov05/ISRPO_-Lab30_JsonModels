## Таблица атрибутов JSON

| Атрибут / настройка | Что делает | Пример |
|---------------------|------------|--------|
| [JsonPropertyName("name")] | Задаёт имя поля в JSON | "name" вместо "Name" |
| [JsonIgnore] | Поле не попадает в JSON | Пароли, внутренние поля |
| PropertyNamingPolicy.CamelCase | Все поля в camelCase | "realName" вместо "RealName" |
| JsonStringEnumConverter | Enum как строка | "Marvel" вместо 0 |
| WriteIndented = true | JSON с отступами | Для удобства при разработке |

## Главные выводы

1. **Сериализация и десериализация** — Сериализация это процесс преобразования C# объекта в JSON-строку для передачи по сети. Десериализация — обратный процесс преобразования JSON-строки обратно в C# объект. Эти механизмы позволяют серверу и клиенту обмениваться структурированными данными.

2. **Зачем нужен [JsonIgnore]** — Этот атрибут исключает поле из JSON-ответа. Реальный пример: поле `InternalNotes` с внутренними заметками разработчика, которое не должно быть видно клиенту, или поле с паролем, которое нельзя отправлять на фронтенд.

3. **Почему enum лучше сериализовывать строкой, а не числом** — Когда enum сериализуется как число, клиент получает "Universe": 0, что неинформативно. При сериализации строкой клиент видит "universe": "Marvel" — сразу понятно, о какой вселенной идёт речь. Это делает API более читаемым и самодокументируемым.

4. **Что изменилось в ответе после настройки CamelCase** — До настройки имена полей были с заглавной буквы: `RealName`, `PowerLevel`. После настройки PropertyNamingPolicy.CamelCase поля стали выглядеть как `realName`, `powerLevel` — это соответствует стандартам JSON и ожиданиям JavaScript-клиентов.
